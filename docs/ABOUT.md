# 关于 SeekU

SeekU 是一个基于 Flutter 的课表管理应用，面向重庆大学学生的真实使用场景设计。当前阶段聚焦 Windows 桌面端，提供本地学期管理、课程查看与编辑、Excel / CSV 导入、AI 识别 PDF 或图片课表、导入预览确认，以及离线持久化。

SeekU 不是简单地“把课表塞进数据库”。它更关心导入前能看清、导入时能校验、导入后能修改；即使使用 AI 识别，也不会盲信模型输出，所有自动导入结果都必须经过预览确认。

## 当前版本

当前版本：`v0.1.1`（pubspec: `0.1.1+8`）

本阶段能力包括：

- 周视图 / 日视图课表展示，启动后默认定位到今日所在教学周的周视图。
- 多学期本地存储，课程数据基于 SQLite 持久化。
- 手动新增、编辑、查看和删除课程。
- Excel / CSV 导入，包含目标学期选择、预览、字段校验、冲突提示和确认保存。
- AI 识别 PDF / 图片课表的初步接入，解析结果进入同一套预览流程。
- 设置页支持用户自行填写并本地保存 Moonshot / Kimi API Key。
- 主页面右下角 AI 核心状态按钮，可检查当前 AI 连接状态。
- 未配置 API Key 时，每次启动软件只弹窗提醒一次；内置试用从首次打开起 5 天内可用。
- 课表设置支持节次数量、非本周课程显示、节次时间和课程颜色管理。
- 全局设置支持主题色、语言占位和字体大小调整。
- 课程详情页支持 CQU-Openlib 自动资源匹配，可展示最佳相关资源链接和候选结果。
- 设置页可查看关于、用户协议、联系方式与开源协议；首次启动会要求确认用户协议。
- 覆盖导入解析、AI JSON 解析、数据库行为、课表规则和基础 Widget 测试。

## 下载方式

普通用户推荐从 GitHub Releases 下载 Windows 安装包：

1. 打开 [SeekU Releases](https://github.com/Tokiperson/seeku/releases)。
2. 选择 `Latest` 或目标版本，例如 `v0.1.1`。
3. 下载 `SeekU-v0.1.1-setup.exe` 并运行安装。

如果 Releases 中暂未看到目标安装包，说明该版本安装包可能仍在发布准备中；开发者可参考 README 从源码运行或构建。

## 功能亮点

### 今日所在周优先

SeekU 启动后默认展示今日所在教学周的周视图，适合打开后直接查看本周课程；主页也保留一键回到今日的入口。

### 多源导入

SeekU 支持 Excel、CSV，并初步接入 PDF / 图片 AI 识别。自动导入不会直接写入正式课表，用户必须先在预览界面确认。

### 本地规则兜底

AI 返回的数据会经过本地解析、字段校验和冲突检测。AI 输出被视为草稿，而不是最终事实。

### 教室校区推断

导入课程时，如果教室字符串首个字符为英文字母，SeekU 会自动作为校区字段。例如 `D1344` 会推断为 `D`。

### 离线优先

核心课表数据保存在本地，AI 能力是增强功能而非硬依赖。即使不配置 AI，SeekU 也可以管理本地课表。

## AI 配置说明

如需使用 AI 识别导入：

1. 打开设置页面。
2. 填写 Moonshot / Kimi API Key。
3. 保存到本地。
4. 回到主页，通过 AI 状态按钮检查连接。

当前规则：

- 用户配置了 API Key 时，优先使用用户配置，并在启动时自动检测连接。
- 未配置 API Key 时，每次启动软件只弹窗提醒一次。
- 内置试用仅允许从首次打开软件起 5 天内使用。
- 超过试用期后，AI 功能必须使用用户自行配置的 API Key。
- AI 输出必须经过预览确认后才能保存为正式课程。

## 当前限制

- AI 导入质量依赖上游模型、网络状态和课表文件清晰度。
- PDF 识别当前优先使用 Kimi 文件解析能力，尚未引入稳定的本地 OCR / 渲染兜底。
- API Key 当前保存在本地偏好设置中，后续应迁移到更安全的本地密钥存储方案。
- Openlib 自动匹配依赖 CQU-Openlib 公开索引与网络状态；无网络或无匹配时只会显示未匹配结果。
- 教务网页导入、账号体系与云同步仍在后续规划中。

## 项目定位

SeekU 不是重庆大学官方产品。它是一个面向学生使用场景的开源工具，请在导入和保存个人课表数据时保持谨慎。
